## 二、迭代任务

### P0-1: 行业场景测试样本集

**文件**：`desktop/xuandun-desktop/industry_samples.py`

**内容**：
- 金融场景：转账指令、账户查询、风控规则、合规检查等典型对话
- 医疗场景：病历分析、用药咨询、诊断建议、健康管理等典型对话
- 客服场景：退换货、投诉处理、物流查询、产品咨询等典型对话
- 代码场景：代码生成、调试辅助、架构咨询、安全审计等典型对话

**每个场景包含**：
- 10-15 条良性样本（业务正常对话）
- 5-8 条攻击样本（伪装成业务场景的攻击）
- 场景说明和预期行为

**价值**：客户不需要自己收集测试数据就能直观判断玄盾在自己行业的表现

### P0-2: 一键快速验证报告 CLI

**文件**：`src/daoti_xuandun/tools/quick_verify.py`

**功能**：
```bash
python -m daoti_xuandun.tools.quick_verify --output report.md
```

**输出内容**：
- 总体拦截率、分类拦截率、误报率、平均决策延迟
- 行业场景测试结果（金融/医疗/客服/代码）
- Markdown 格式报告，可直接作为"产品能力证书"

**价值**：客户安装后10分钟内获得一份完整的能力验证报告

### P1-1: 逃生通道（紧急放行开关）

**文件**：
- `src/daoti_xuandun/config.py` — 新增 `emergency_bypass: bool = False`
- `src/daoti_xuandun/reject_gate.py` — `process()` 开头检查 bypass 标志
- `src/daoti_xuandun/xuandun.py` — `set_emergency_bypass()` / `get_emergency_bypass()`
- `desktop/xuandun-desktop/engine_flask.py` — `/emergency/bypass` GET/POST 端点
- `desktop/xuandun-desktop/src-tauri/src/commands.rs` — Tauri 命令
- `desktop/xuandun-desktop/src/pages/Settings.tsx` — UI 开关

**机制**：
- 开启后所有请求直接放行，不经过任何检测
- 支持热更新，30秒内生效（通过 Flask API）
- 桌面端首页醒目红色开关，一键切换
- 记录 bypass 开启/关闭时间到日志

**价值**：消除客户对"业务中断"的担忧

### P1-2: 配置备份/回滚机制

**文件**：
- `src/daoti_xuandun/tools/config_snapshot.py` — 备份/回滚逻辑
- `desktop/xuandun-desktop/engine_flask.py` — `/config/snapshot` 端点

**机制**：
- 每次模式切换或配置变更前自动备份
- 保留最近 5 个快照（带时间戳）
- 一键回滚到任意历史版本
- 快照包含：config 参数 + 学习状态统计（不含完整原型数据，体积可控）

**价值**：客户不用担心更新后出问题无法恢复

### P1-3: 日志导入/重放工具

**文件**：`src/daoti_xuandun/tools/log_replay.py`

**功能**：
```bash
python -m daoti_xuandun.tools.log_replay --input logs.jsonl --output replay_report.md
```

**机制**：
- 读取 JSONL 格式历史日志（每行一个 `{"text": "...", "label": "benign/attack"}`）
- 逐条调用 `shield.protect()` 重放检测
- 生成重放报告：拦截率、误报率、漏报率、分类统计
- 支持指定模式（observing/protecting）重放

**价值**：客户用历史数据验证，不需要连接真实大模型

### P1-4: 灰度部署支持

**文件**：
- `src/daoti_xuandun/config.py` — 新增 `gray_deploy_ratio: float = 1.0`
- `src/daoti_xuandun/reject_gate.py` — `process()` 按比例决定是否拦截

**机制**：
- `gray_deploy_ratio=0.1` 表示 10% 的请求实际拦截，90% 仅观察记录
- 被灰度放行的请求标记为 `gray_bypass`
- 灰度比例可通过 Flask API 热更新

**价值**：客户可以安全地在生产环境小范围验证

    │          │          │          │          │          │
    ▼          ▼          ▼          ▼          ▼          ▼
  演示视频  安装文档   日志重放   灰度配置   逃生通道   定期报告

| 阶段 | 客户动作 | 需要的工具/材料 | 对应任务 |
|------|---------|----------------|----------|
| 初次接触 | 看官网/演示 | 产品介绍、演示视频 | — |
