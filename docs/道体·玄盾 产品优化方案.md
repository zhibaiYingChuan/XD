# 道体·玄盾 产品优化方案（基于“观察→学习→自动切换”思路）

## 一、当前状态回顾

**你已确认的核心逻辑：**

玄盾应该先“听着”——接入网关后不拦截任何请求，只旁听并记录所有的日常对话，自动学习模型所在环境的“正常”语言模式，积累足够样本后自动切换到拦截模式。

**你已确认不做的：**
- 不强制预热
- 不需要准备各行业预热文本
- 不需要用户手动配置规则

**你已确认内置的：**
- 内置攻击类型样本（提示注入、越狱、编码攻击等），让玄盾一开始就知道“什么是坏”

### 2.2 洛书空间的两个“边界”

| 边界 | 内容 | 来源 | 是否固定 |
|------|------|------|----------|
| **安全域** | 正常请求的聚类中心 | 观察模式中积累 | 持续扩展 |
| **攻击域** | 攻击样本的聚类中心 | 内置攻击样本 + 实际拦截积累 | 持续扩展 |


## 三、功能模块改造

### 3.1 核心引擎改造（reject_gate.py）

#### 新增：观察模式

```python
class EndogenousDomainAwareness:
    def __init__(self, config):
        self.mode = "observing"  # "observing" | "protecting"
        self.sample_count = 0
        self.min_samples_for_switch = 1000  # 可配置
        
    def process(self, raw_input):
        # 1. 计算洛书距离（无论什么模式）
        dist, ... = self._compute_distance(raw_input)
        
        # 2. 学习：每条请求都记录到洛书空间
        self._learn_from_request(raw_input, dist)
        
        # 3. 自动切换判断
        if self.mode == "observing" and self.sample_count >= self.min_samples_for_switch:
            self.mode = "protecting"
            self._emit_status_change("LEARNING_COMPLETE")
        
        # 4. 决策
        if self.mode == "observing":
            # 观察模式：放行所有请求，但记录分析结果
            return Decision.PASS, TrustLevel.LOW, dist
        else:
            # 拦截模式：正常决策
            return self._make_decision(dist, ...)
```

### 4.2 学习进度的计算

```python
def _get_learning_progress(self):
    """返回0-1之间的进度值"""
    if self.sample_count >= self.min_samples_for_switch:
        return 1.0
    return self.sample_count / self.min_samples_for_switch
```

### 4.3 自动切换的触发

```python
def _check_auto_switch(self):
    """每次process后调用"""
    if self.mode != "observing":
        return
    if self.sample_count >= self.min_samples_for_switch:
        self.mode = "protecting"
        self._emit_event("MODE_SWITCHED", {
            "from": "observing",
            "to": "protecting",
            "samples_collected": self.sample_count
        })
```

## 七、总结

| 原问题 | 解决方案 |
|--------|----------|
| 用户需要预热样本 | 不需要，系统自动学习 |
| 需要准备行业文本 | 不需要，内置攻击样本 + 自动学习 |
| 安装后没反应 | 观察模式下状态条显示学习进度，“它在学着保护你” |
| 不知道什么时候开始保护 | 自动切换时通知，进度条显示预计时间 |
| 误拦 | 观察模式保证0误拦，拦截模式有学习基础 |
| 运维人员如何验证 | 仪表盘显示学习进度、原型统计、模拟拦截日志 |

**一句话总结：**

> 用户接入玄盾 → 玄盾自动旁听正常对话 → 学到足够多的“正常”后自动开启保护 → 内置攻击样本让它一开始就知道什么是“坏”。

这是“活性防护”在B端场景下的最佳落地形态。
