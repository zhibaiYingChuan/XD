# 道体·玄盾 端到端验证清单

> **文档用途**：供 QA / 实地验证人员对"道体·玄盾"LLM 防火墙产品进行端到端功能验证。
> **执行方式**：按顺序执行各测试用例，逐项填写"实际结果"与"通过/失败"列。
> **版本**：v1.0 ｜ **日期**：2026-07-11 ｜ **产品版本**：引擎 v1.2.0 / 桌面端 v1.2.0

---

## 一、测试环境与前置准备

### 1.1 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11、macOS 12+、Ubuntu 20.04+（三选一） |
| 桌面端 | 已安装 Tauri 三端应用（xuandun-desktop） |
| 引擎地址 | `http://127.0.0.1:18765`（桌面端启动后自动拉起） |
| 测试工具 | curl / Postman / 浏览器开发者工具 |
| 测试账号 | 本地引擎无需账号，直接调用 API |

### 1.2 引擎 API 端点清单（实地验证用）

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/health` | 健康检查 |
| GET | `/status` | 引擎运行状态（含 learning_mode、sample_count） |
| GET | `/learning/status` | 学习状态详情（模式、进度、原型数、模拟拦截预览） |
| GET | `/learning/details` | 原型统计摘要与典型输入示例 |
| POST | `/protect` | 文本检测（注意：实际端点为 `/protect`，非 `/detect`） |
| POST | `/mode/switch` | 手动切换 观察/保护 模式，Body: `{"mode":"observing"\|"protecting"}` |
| POST | `/set-mode` | 切换防御层级，Body: `{"mode":"high_security"\|"balanced"\|"low_false_positive"}` |
| POST | `/simulation/run` | 模拟测试，Body: `{"mode":"quick"\|"full"\|"custom", ...}` |
| POST | `/warmup` | 领域预热，Body: `{"safe_texts":[...], "attack_texts":[...]}` |
| GET | `/ping` | 心跳 |

### 1.3 关键参数（默认值）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `enable_observing_mode` | `True` | 启动默认进入观察模式 |
| `min_samples_for_switch` | `1000` | 自动切换到保护模式所需最小正常样本数 |
| `enable_builtin_attacks` | `True` | 启动时加载内置攻击原型（约36条种子） |
| 默认防御层级 | `balanced`（STANDARD） | 兼顾安全与可用性 |
| 模拟测试样本库 | 200+ 条攻击 + 30 条良性 | 覆盖 OWASP LLM Top 10 六大类 |

### 1.4 测试前重置状态

每个测试场景开始前，建议通过设置页"重启引擎"或删除引擎数据目录恢复初始状态，确保 `sample_count=0`、`mode=observing`。
