# 道体·玄盾 桌面端 UI/UX 设计规范（完整版）

> **版本**：v2.0  
> **适用产品**：道体·玄盾 安全管理控制台（桌面端）  
> **设计目标**：为 AI 安全产品提供一套专业、可落地、视觉统一的设计系统


## 一、设计哲学与原则

### 1.1 品牌心智

玄盾的品牌体验应传递以下三个核心感知：

| 感知维度 | 描述 | 视觉表达 |
|---------|------|----------|
| **安全感** | 系统稳定可靠，值得信赖 | 深色基调、沉稳的蓝、明确的成功/失败状态 |
| **科技感** | 先进但不浮夸，智能但不黑盒 | 微光效、数据可视化、干净的排版 |
| **智慧感** | 能思考、会学习、懂业务 | 动态元素、成长性可视化（如原型库演化图） |

### 1.2 设计原则

- **安全优先**：危险操作需二次确认，关键状态变更必须可见
- **信息分层**：从宏观概览到微观详情，层层递进，避免信息过载
- **可解释性**：系统的每个决策都应该能被用户理解（为什么拦截？为什么放行？）
- **效率至上**：安全分析师的操作路径应≤3步到达任何功能
- **一致性**：全平台统一的交互语言和视觉语言

| 色名 | 色值 | 用途 | 使用比例 |

| 道体蓝悬停 | `#3B7AFF` | 按钮悬停、链接悬停 | — |

| 卡片黑（三级背景） | `#1C2330` | 表格行、输入框背景、悬浮卡片 | — |

#### 状态色

| 状态 | 色值 | 用途 |
|------|------|------|
| **安全/通过** | `#00D4AA` | 请求通过、系统安全、成功状态 |
| **警告/可疑** | `#F5A623` | 低风险告警、待审核、部分异常 |
| **危险/拦截** | `#E54D4D` | 拦截、高风险告警、错误状态 |
| **信息/中性** | `#6B7A8F` | 普通信息、辅助文本、不可用状态 |
| **重点关注** | `#FF6B35` | 需要人工介入的事件、紧急操作 |

#### 状态色应用规则

- 每个独立卡片/模块内，状态色使用不超过**3种**，避免视觉混乱
- 状态色应配合图标使用，色盲用户仍需通过图标识别状态
- 危险操作（如删除、清空日志）应使用危险色作为按钮主色

### 2.5 品牌纹理与光效

#### 微光效

- 页面顶部可有一条微弱的品牌色渐变光晕（`#2B5FD7` 20% 透明度径向渐变）
- 卡片悬停时可添加极淡的蓝色发光边框（`box-shadow: 0 0 30px rgba(43,95,215,0.05)`）
- 关键数据卡片（如 KPI 卡片）可带有微弱的品牌色底光

#### 网格背景

- 主背景可叠加极淡的网格线（`rgba(255,255,255,0.02)`，间距 24px）
- 增加屏幕的科技感深度，不干扰内容阅读

| 尺寸 | 高度 | 内边距 | 字号 | 圆角 |
|------|------|--------|------|------|
| 小 | 32px | 0 16px | 13px | 6px |

- **默认**：正常显示

- **加载**：显示加载动画，文字变淡，不可操作
- **禁用**：透明度降低，不可操作，显示禁用提示

## 六、交互与动效规范

### 6.1 页面转场

| 转场类型 | 时长 | 缓动函数 | 描述 |

| 新数据到达 | 卡片高亮闪烁 | 数据更新时 |

## 八、暗色模式

### 8.1 暗色模式规范

玄盾桌面端以暗色模式为默认（也是唯一）主题。所有色彩规范已在前面章节定义，此处强调暗色模式下的特殊考虑：

- **避免纯黑**：主背景 `#0B0E14` 略微偏蓝，减少纯黑带来的压抑感
- **发光控制**：品牌色发光仅用于焦点元素，不过度使用
- **对比度**：所有文本与背景对比度 ≥ 4.5:1（WCAG AA 标准）

## 十、参赛展示建议

### 10.1 展示故事线

建议以“一个安全主管的早晨”作为叙事线索：

1. **打开玄盾** → 仪表盘（3秒内掌握全局安全态势）
2. **发现异常** → 看到拦截率突然升高，点击进入日志
3. **深入调查** → 筛选高风险日志，点击查看详情
4. **理解原因** → 看到风险评分详情和决策原因
5. **采取行动** → 调整防护规则或添加白名单
6. **验证效果** → 回到仪表盘确认系统恢复正常

### 10.2 展示要点

- **强调差异点**：在日志详情页突出显示“决策原因”和“安全原型库匹配”，这是玄盾区别于传统 WAF 的核心竞争力
- **数据可视化**：仪表盘的图表是视觉焦点，确保图表设计专业、清晰
- **微交互**：按钮悬停、卡片上浮、弹窗动画等细节是评分关键
- **一致性**：所有页面的颜色、字体、组件风格保持一致

### 10.3 评审关注点

| 评审维度 | 权重 | 展示重点 |
|---------|------|----------|
| 视觉设计 | 30% | 色彩、排版、图标、整体美感 |
| 交互设计 | 25% | 操作流程、反馈、动效 |
| 信息架构 | 20% | 页面布局、信息层级、可寻性 |
| 产品创新 | 15% | 差异化功能展示（活性学习、可解释性） |
| 设计系统 | 10% | 组件库、设计规范、一致性 |
