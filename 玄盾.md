> **版本**：v1.0\
> **状态**：规约冻结\
> **目标**：为AI代码助手提供无歧义、可执行的安全模块开发规约。

## 1. 产品概述

### 1.1 定位

道体·玄盾是一个**轻量级AI运行时安全网关**，可为任意大模型API、AI应用提供四个可叠加的安全防护层。

### 1.2 核心价值

- 主动防御未知攻击（提示注入、模型逆向、对抗样本）
- 动态混淆，防止模型权重被白盒提取
- 运行时异常行为检测

### 1.3 交付形态

- **核心库**：Python wheel包，闭源（核心二进制），提供纯Python调用接口。
- **依赖**：Python ≥ 3.10, numpy, torch (可选，仅用于向量计算)
- **无网络请求**：所有检测均在本地完成。

### 1.4 四个安全机制（递进式顺序）

| 阶段 | 机制      | 输入        | 输出     | 说明         |
| :- | :------ | :-------- | :----- | :--------- |
| 1  | 主动拒绝门禁  | 原始输入向量/文本 | 通过/拒绝  | 快速过滤未知域    |
| 2  | 动态阴阳壳   | 通过门禁的向量   | 混淆后的向量 | 固定递归层，防御逆向 |
| 3  | 古文符号映射  | 混淆后向量     | 离散符号序列 | 破坏梯度传递     |
| 4  | 时序一致性校验 | 符号序列 + 状态 | 通过/告警  | 检测重放及节律异常  |

> 用户可选择启用任意子集，默认全启用。

## 3. 模块一：主动拒绝门禁

### 3.1 功能规格

- **输入**：任意用户输入 `raw_input: Union[str, Vector]`
- **处理**：
  1. 将输入转换为固定维度的特征向量 `feat`（如使用内置轻量embedder或要求调用者提供）。
  2. 使用一个预训练的**域分类器**（多分类 softmax）计算 `feat` 属于各个已知域的概率。
  3. 若最大概率 < `domain_threshold`，则拒绝；否则输出 `(feat, domain_id)`。
- **输出**：`Decision` 枚举（`PASS` / `REJECT`）及通过时的附带数据。

### 3.2 接口定义

```python
class RejectGate:
    def __init__(self, config: XuanDunConfig, domain_classifier: Any):
        """
        domain_classifier: 实现了 predict_proba(feat: np.ndarray) -> np.ndarray
                           返回形状 (num_domains,) 的概率。
        """
        ...
    
    def process(self, raw_input: Union[str, Vector]) -> Tuple[Decision, Optional[Vector], Optional[int]]:
        """
        返回 (decision, vector, domain_id)
        """
        ...
```

### 3.3 域分类器预训练要求（由用户提供或道体内置基础版）

- 输入维度 = `hidden_dim`
- 输出维度 = 已知域的数量（配置中的 `known_domains`）
- 采用 softmax + 交叉熵训练，达到 ≥90% 准确率。

## 4. 模块二：动态阴阳壳

### 4.1 功能规格

- **输入**：来自模块一的向量 `x` (shape: `hidden_dim`)
- **处理**：
  - 固定随机种子（由 `shell_key` 派生）初始化权重矩阵 `W_f, U_f, b_f, W_b, U_b, b_b`（与之前 Rust 实现类似）。
  - 执行 `t_iter` 次双向递归：
    - 正向：`h_f[i] = tanh(W_f @ h_f[i-1] + U_f @ h_b[i] + b_f)` （下标表示层）
    - 逆向：`h_b[i] = tanh(W_b @ h_b[i+1] + U_b @ h_f[i] + b_b)`
  - 取最顶层的逆向状态作为输出 `y`。
- **输出**：混淆后向量 `y` (shape: `hidden_dim`)

### 4.2 约束

- **权重固定**：在初始化后不再变化（非训练模式）。
- **纯 numpy 实现**：不依赖 torch（降低依赖）。
- **数值稳定性**：tanh 使用 `np.tanh`。

### 4.3 接口定义

```python
class DynamicShell:

        # 根据 hidden_dim, num_layers, shell_key 初始化所有权重矩阵
        self._init_weights()
    
    def transform(self, x: np.ndarray) -> np.ndarray:
        """
        x: shape (hidden_dim,)
        returns: y shape (hidden_dim,)
        """
        ...
```

### 4.4 权重初始化算法

```python
def _init_weights(self):
    rng = np.random.default_rng(seed=int.from_bytes(self.shell_key, 'little'))
    limit = 0.1
    self.W_f = rng.uniform(-limit, limit, (self.dim, self.dim))
    self.U_f = rng.uniform(-limit, limit, (self.dim, self.dim))
    self.b_f = np.zeros(self.dim)
    self.W_b = rng.uniform(-limit, limit, (self.dim, self.dim))
    self.U_b = rng.uniform(-limit, limit, (self.dim, self.dim))
    self.b_b = np.zeros(self.dim)
```

### 4.5 单元测试

```python
    config2 = XuanDunConfig(shell_key=b'key2')

    x = np.random.randn(hidden_dim)
    y1, y2 = shell1.transform(x), shell2.transform(x)
    assert not np.allclose(y1, y2)   # 不同密钥应产生完全不同结果
```

## 5. 模块三：古文符号映射

### 5.1 功能规格

- **输入**：混淆后向量 `y` (shape: `hidden_dim`)
- **处理**：
  1. 使用 `mapping_key` 生成一个随机的双射 `π: [0, hidden_dim) -> [0, symbol_table_size)` （实际映射为每个维度选择一个符号，但为增强混淆，可采用“赢家通吃”或“分桶”）。
  2. 更具体实现：将 `y` 的每个分量值映射到符号表中的一个符号，形成符号序列。
  3. 推荐做法：计算 `y` 的 argmax 将向量压缩为一个符号，但原意是破坏梯度，可对每个维度独立映射。
  4. 为平衡安全与信息量，采用**向量到符号序列**：`symbols[i] = π( floor( (y[i] + 1) / 2 * symbol_table_size ) )`，但 π 是一个随机置换表。
- **输出**：符号序列 `sym_seq: List[int]`，长度 = `hidden_dim`。

### 5.2 接口定义

```python
class AncientSymbolMapper:
    def __init__(self, config: XuanDunConfig):
        # 生成置换表 perm: List[int] 长度 symbol_table_size
        rng = np.random.default_rng(seed=int.from_bytes(config.mapping_key, 'little'))
        perm = np.arange(config.symbol_table_size)
        rng.shuffle(perm)
        self.perm = perm
        self.table_size = config.symbol_table_size
    
    def map(self, vec: np.ndarray) -> List[int]:
        # 将 vec 的每个值 [-1,1] 映射到 [0, table_size-1]
        idx = np.floor((vec + 1) / 2 * self.table_size).astype(int)
        idx = np.clip(idx, 0, self.table_size - 1)
        return [int(self.perm[i]) for i in idx]
```

## 6. 模块四：时序一致性校验

### 6.1 功能规格

- **输入**：符号序列 `sym_seq` （来自模块三）以及可选的会话ID
- **状态维护**：
  - 为每个会话（例如API key+用户IP）维护一个滑动窗口，记录最近 `max_window_size` 个符号序列的**特征向量**（如序列的直方图或嵌入）。
  - 计算新序列与历史窗口的马氏距离，若超过 `anomaly_threshold` 则触发告警。
- **处理流程**：
  1. 从 `sym_seq` 提取特征 `feat_seq`（例如长度为 `symbol_table_size` 的直方图，归一化）。
  2. 若窗口未满，加入并返回 `PASS`。
  3. 若已满，计算当前特征与窗口内所有特征的均值向量及协方差矩阵的逆，计算马氏距离。
  4. 若距离 > `anomaly_threshold` → 告警，可选择拒绝或记录。
  5. 更新窗口（FIFO）。
- **输出**：`TimingDecision` (`PASS`, `WARN`, `REJECT`)

### 6.2 接口定义

```python
class TimingConsistencyChecker:
    def __init__(self, config: XuanDunConfig):
        self.window_size = config.max_window_size
        self.threshold = config.anomaly_threshold
        self.state = {}   # session_id -> deque of feature vectors
    
    def check(self, sym_seq: SymbolSeq, session_id: str) -> Tuple[TimingDecision, float]:
        """
        返回 (decision, distance)
        """
        ...
    
    def _extract_feature(self, sym_seq: SymbolSeq) -> np.ndarray:
        hist = np.zeros(self.table_size)
        for s in sym_seq:
            hist[s] += 1
        return hist / len(sym_seq)   # 归一化直方图
```

### 6.3 马氏距离实现

```python
def mahalanobis(x, mean, cov_inv):
    delta = x - mean
    return np.sqrt(delta.T @ cov_inv @ delta)
```

> 注意：使用正则化协方差矩阵（`np.linalg.pinv` 或加小单位矩阵）防止奇异。

```python
class XuanDun:
    def __init__(self, config: XuanDunConfig, domain_classifier: Optional[Any] = None):

        self.dynamic_shell = DynamicShell(config) if config.enable_dynamic_shell else None

        self.timing_checker = TimingConsistencyChecker(config) if config.enable_timing_check else None

        1. 主动拒绝门禁
        2. 动态阴阳壳
        3. 古文符号映射
        4. 时序校验
        如果任一阶段返回REJECT，立即停止并返回拒绝结果。
        """
        ...
```

```python
class ProtectResult:
```

## 8. 递进式开发与AI约束

### 8.1 开发顺序

1. **基础结构**：定义 `XuanDunConfig`，空壳类。
2. **模块一（主动拒绝门禁）**：实现、测试。
3. **模块二（动态阴阳壳）**：实现、测试。
4. **模块三（古文符号映射）**：实现、测试。
5. **模块四（时序一致性校验）**：实现、测试。
6. **集成类**：串联所有模块，端到端测试。

### 8.2 AI代码助手强制约束

- 所有公共函数必须包含完整的 docstring（参数、返回值、异常）。
- 每个函数上方必须注释对应的规约章节号，例如 `# Implements §4.3 transform`。
- 禁止使用 `eval`、`exec` 或动态导入未知模块。
- 禁止硬编码任何路径、URL、IP地址。
- 所有随机性必须由配置中的 `key` 种子驱动，确保可重现。
- 单元测试覆盖率 ≥ 90%。

### 8.3 验证方式

- 运行 `pytest tests/` 所有测试必须通过。
- 提供 `demo.py` 展示端到端防护流程（包含模拟攻击用例）。
